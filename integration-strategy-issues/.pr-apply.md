## What

Implements the apply path: given a referenced `WorkspaceIntegrationStrategy`,
resolve its templates and merge the resulting pod modifications (sidecars, init
containers, volumes, volume mounts, env) into the workspace deployment.

- `DeploymentBuilder.BuildDeploymentWithStrategies` + `ApplyIntegrationStrategyToDeployment`
  and the merge / resource-fetch helpers
- `ResourceManager.GetIntegrationStrategyForWorkspace`, with the
  `integrationStrategy` threaded through resource manager → state machine →
  workspace controller
- `ray.io/rayclusters` `get;list` RBAC for the resourceLookup fetch
- Unit + envtest coverage for the merge logic and the running reconcile path

## Scope

The "transform a deployment with a strategy" half of the controller. The watch /
label-sync / reverse-lookup half is a separate PR.
