## What

Adds the `WorkspaceIntegrationStrategy` API type and the `IntegrationStrategyRef`
on `Workspace.spec` (name/value parameter list + `ParametersMap` helper), the
generated deepcopy and CRD manifests, and the annotated Ray integration strategy
sample as the worked example of the API.

This is the foundation of the WorkspaceIntegrationStrategy feature — an
admin-installed, reusable strategy that declaratively injects runtime
capabilities (sidecars, volumes, env vars) into a workspace pod with dynamic
template resolution.

## Scope

This PR is the **API surface only** — types, CRDs, and the sample. It builds and
passes on its own. The resolver, controller wiring, and e2e tests follow in
separate PRs.

## Dependencies

None. This is the base PR; the others depend on it.
